OpenCLI Admin is a full-stack, self-hosted data collection platform that lets you visually manage scraping jobs across 20+ platforms — including Xiaohongshu, Bilibili, Weibo, Twitter/X, Hacker News, and more. Built on top of [opencli](https://github.com/jackwener/opencli), it adds a web dashboard, AI-powered content tagging, distributed node management, and push notification delivery — all without writing code.

## What you can build

<Steps>
  <Step title="Connect your data sources">
    Add sources from any of five channel types: opencli (browser-based), RSS feeds, REST APIs, web scrapers, or generic CLI commands. Each source maps to a specific platform or endpoint.
  </Step>
  <Step title="Schedule automated collection">
    Create cron-based schedules (every N minutes, hours, days, or weeks) with timezone support. Bind each schedule to a specific agent node or let the system auto-route.
  </Step>
  <Step title="Process with AI">
    Configure an AI agent using Claude, OpenAI, DeepSeek, Kimi, GLM, or Ollama. After each collection run, the pipeline automatically summarizes and tags every new record.
  </Step>
  <Step title="Push to your team">
    Send notifications to Feishu, DingTalk, WeCom, Email, or any Webhook endpoint. Trigger on new records, AI completion, or task failures.
  </Step>
</Steps>

## Supported platforms

| Category | Platforms |
|----------|-----------|
| Chinese social | Xiaohongshu, Bilibili, Zhihu, Weibo, V2EX, Snowball |
| Chinese news/finance | Sina Finance 7×24, Ctrip, Boss Zhipin |
| Global news | Hacker News, BBC, Reuters, Bloomberg, arXiv, Wikipedia |
| Global social | Twitter/X, Reddit, YouTube, LinkedIn, Yahoo Finance, Barchart |

<Note>
  Platforms requiring login (Xiaohongshu, Bilibili, Weibo, Twitter/X, LinkedIn, YouTube) need a one-time manual login via the browser. Public sources like Hacker News, BBC, and RSS feeds work without authentication.
</Note>
