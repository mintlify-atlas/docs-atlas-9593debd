---
title: "Sources API: Create and Manage Data Source Channels"
description: "Create, read, update, delete, and test connectivity for data sources across opencli, RSS, API, web scraper, and CLI channel types."
---

Data sources are the root configuration objects in OpenCLI Admin. Each source defines a channel type and its associated configuration. Schedules, tasks, and records all reference a source. The supported channel types are `opencli`, `rss`, `api`, `web_scraper`, and `cli`.

---

## List sources

`GET /api/v1/sources`

Returns a paginated list of data sources. All query parameters are optional.

### Query parameters

<ParamField query="enabled" type="boolean">
  Filter by enabled state. Omit to return both enabled and disabled sources.
</ParamField>

<ParamField query="channel_type" type="string">
  Filter by channel type. One of `opencli`, `rss`, `api`, `web_scraper`, `cli`.
</ParamField>

<ParamField query="page" type="integer" default="1">
  Page number. Must be â‰Ą 1.
</ParamField>

<ParamField query="limit" type="integer" default="20">
  Items per page. Accepts 1â€“100.
</ParamField>

### Response

<ResponseField name="data" type="DataSourceRead[]">
  Array of source objects.
  <Expandable title="DataSourceRead properties">
    <ResponseField name="id" type="string">Unique source ID (UUID).</ResponseField>
    <ResponseField name="name" type="string">Display name.</ResponseField>
    <ResponseField name="description" type="string">Optional description.</ResponseField>
    <ResponseField name="channel_type" type="string">Channel type: `opencli` | `rss` | `api` | `web_scraper` | `cli`.</ResponseField>
    <ResponseField name="channel_config" type="object">Channel-specific configuration object.</ResponseField>
    <ResponseField name="ai_config" type="object">Optional AI processing configuration.</ResponseField>
    <ResponseField name="enabled" type="boolean">Whether the source is active.</ResponseField>

```bash
curl http://localhost:8031/api/v1/sources?enabled=true&channel_type=rss&page=1&limit=20
```

## Create source

`POST /api/v1/sources`

Creates a new data source. Returns `201 Created` on success.

### Request body

<ParamField body="name" type="string" required>
  Display name. 1â€“255 characters.
</ParamField>

<ParamField body="channel_type" type="string" required>
  Channel type. One of `opencli`, `rss`, `api`, `web_scraper`, `cli`.
</ParamField>

<ParamField body="channel_config" type="object">
  Channel-specific configuration. Defaults to `{}`.
</ParamField>

<ParamField body="description" type="string">
  Optional human-readable description.
</ParamField>

<ParamField body="ai_config" type="object">
  Optional AI processing configuration linked to an AI agent.
</ParamField>

<ParamField body="enabled" type="boolean" default="true">
  Whether the source starts enabled.
</ParamField>

<ParamField body="tags" type="string[]">
  User-defined tags. Defaults to `[]`.
</ParamField>

```bash
curl -X POST http://localhost:8031/api/v1/sources \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Hacker News RSS",
    "channel_type": "rss",
    "channel_config": { "url": "https://news.ycombinator.com/rss" },
    "enabled": true,
    "tags": ["tech", "news"]
  }'
```

## Get source

`GET /api/v1/sources/{source_id}`

Returns the full detail of a single source, including connectivity status fields from the last test.

### Path parameters

<ParamField path="source_id" type="string" required>
  Source UUID.
</ParamField>

### Response

<ResponseField name="data" type="DataSourceDetail">
  Extends `DataSourceRead` with connectivity fields.
  <Expandable title="additional properties">
    <ResponseField name="connectivity_ok" type="boolean">
      Result of the last connectivity test. `null` if never tested.
    </ResponseField>
    <ResponseField name="connectivity_errors" type="string[]">
      Error messages from the last connectivity test.
    </ResponseField>
  </Expandable>
</ResponseField>

```bash
curl http://localhost:8031/api/v1/sources/src_abc123
```

## Update source

`PATCH /api/v1/sources/{source_id}`

Partially updates a source. Only fields included in the request body are changed.

### Path parameters

<ParamField path="source_id" type="string" required>
  Source UUID.
</ParamField>

### Request body

<ParamField body="name" type="string">
  Updated display name. 1â€“255 characters.
</ParamField>

<ParamField body="description" type="string">
  Updated description.
</ParamField>

<ParamField body="channel_config" type="object">
  Replacement channel configuration object.
</ParamField>

<ParamField body="ai_config" type="object">
  Replacement AI processing configuration.
</ParamField>

<ParamField body="enabled" type="boolean">
  Enable or disable the source.
</ParamField>

<ParamField body="tags" type="string[]">
  Replacement tags list.
</ParamField>

```bash
curl -X PATCH http://localhost:8031/api/v1/sources/src_abc123 \
  -H "Content-Type: application/json" \
  -d '{ "enabled": false }'
```

## Delete source

`DELETE /api/v1/sources/{source_id}`

Permanently deletes a source and all its associated tasks, schedules, and records (cascade delete).

### Path parameters

<ParamField path="source_id" type="string" required>
  Source UUID.
</ParamField>

```bash
curl -X DELETE http://localhost:8031/api/v1/sources/src_abc123
```

## Test source connectivity

`POST /api/v1/sources/{source_id}/test`

Runs a connectivity check for the source without triggering a full collection. Returns a `connected` boolean and any error messages.

### Path parameters

<ParamField path="source_id" type="string" required>
  Source UUID.
</ParamField>

### Response

<ResponseField name="data" type="object">
  <Expandable title="properties">
    <ResponseField name="connected" type="boolean">
      `true` if the channel is reachable and the configuration is valid.
    </ResponseField>
    <ResponseField name="errors" type="string[]">
      List of error messages if `connected` is `false`.
    </ResponseField>
  </Expandable>
</ResponseField>

```bash
curl -X POST http://localhost:8031/api/v1/sources/src_abc123/test
```
