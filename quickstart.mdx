title: "Quickstart: Run Your First OpenCLI Admin Deployment"

This guide gets you from zero to a working dashboard as quickly as possible. There are two paths: **Docker** (recommended for most users — no local Python or Node.js required) and **Native Shell** (best for development or when you already have Chrome and Python set up locally). Both paths expose the same dashboard on port `8030` and API docs on port `8031`.

<Tabs>
  <Tab title="Docker (recommended)">

    Docker Compose pulls pre-built images from Docker Hub and starts the API, frontend, and a built-in agent container in one command.

    ### Prerequisites

    - Docker Engine 24+ and Docker Compose v2
    - Chrome installed on the host machine (only required for the standard agent image; see the tip below)

    ### Steps

    <Steps>
      <Step title="Copy the environment file">
        Clone or download the repository, then copy the example environment file:

        ```bash
        cp .env.example .env
        ```
      </Step>

      <Step title="Set a secure SECRET_KEY">
        Open `.env` and replace the placeholder value:

        ```bash
        SECRET_KEY=change-me-in-production-use-long-random-string
        ```

        Replace it with a long random string before exposing the service to any network.

        <Warning>
          The default `SECRET_KEY` value is public. Always replace it before running in any environment beyond strictly localhost.
        </Warning>
      </Step>

      <Step title="Start host Chrome with remote debugging (standard image only)">
        The default agent image (~100 MB) connects to Chrome running on your Docker host via `host.docker.internal`. Start Chrome with CDP enabled before running Compose:

        <CodeGroup>

        ```bash macOS
        /Applications/Google\ Chrome.app/Contents/MacOS/Google\ Chrome \
            --remote-debugging-port=9222 --remote-debugging-address=0.0.0.0 \
            --no-first-run --no-default-browser-check &
        ```

        ```bash Linux
        google-chrome \
            --remote-debugging-port=9222 --remote-debugging-address=0.0.0.0 \
            --no-first-run --no-default-browser-check &
        ```

        </CodeGroup>

        <Tip>
          To skip this step, use the chrome-embedded agent image by setting `CHROME_SUFFIX=-chrome` in `.env`. That image is ~450 MB but is fully self-contained — no host Chrome required.
        </Tip>
      </Step>

      <Step title="Start all services">
        ```bash
        docker compose up -d
        ```

        Docker pulls `xjh1994/opencli-admin-{api,frontend,agent}:0.3.6` from Docker Hub on first run.
      </Step>

      <Step title="Open the dashboard">
        Once the API health check passes (usually within 20 seconds):

        | Service | URL |
        |---------|-----|
        | Dashboard | http://localhost:8030 |
        | API docs | http://localhost:8031/docs |
        | Agent noVNC | http://localhost:3010 |
      </Step>

      <Step title="Log in to collection platform accounts">
        Platforms like Xiaohongshu, Bilibili, Twitter/X, and LinkedIn require a browser login before collection works. Open the noVNC viewer to log in from within the agent's browser:

        - **Agent 1:** http://localhost:3010
        - **Agent 2:** http://localhost:3011
        - **Agent 3:** http://localhost:3012

        Login state is saved in an isolated Docker volume per agent instance, so you only need to do this once per agent.
      </Step>
    </Steps>

  </Tab>
  <Tab title="Native Shell">

    The native shell path runs the backend and frontend directly on your machine, reusing your existing Chrome installation. Both the backend (uvicorn `--reload`) and the frontend (Vite HMR) run in hot-reload mode, making it ideal for development.

    ### Prerequisites

    - Python 3.11+
    - Node.js 18+
    - Google Chrome or Chromium installed
    - [opencli Browser Bridge extension](https://github.com/jackwener/opencli/blob/main/README.zh-CN.md#playwright-mcp-bridge-%E6%89%A9%E5%B1%95%E9%85%8D%E7%BD%AE) installed in Chrome

    ### Steps

    <Steps>
      <Step title="Copy the environment file">
        ```bash
        cp .env.example .env
        ```
      </Step>

      <Step title="Set a secure SECRET_KEY">
        Edit `.env` and replace the placeholder:

        ```bash
        SECRET_KEY=change-me-in-production-use-long-random-string
        ```
      </Step>

      <Step title="Run the startup script">
        ```bash
        ./start.sh
        ```

        `start.sh` automatically handles everything in sequence:

        - Creates a Python virtual environment in `.venv`
        - Installs all backend Python dependencies
        - Checks for `opencli` and offers to install it via npm if missing
        - Initializes the SQLite database and runs migrations
        - Starts Chrome with CDP on port `9222` using the profile at `~/.opencli-admin/chrome-profile/`
        - Starts the FastAPI backend with hot reload on port `8031`
        - Installs frontend npm dependencies and starts the Vite dev server with HMR on port `8030`
      </Step>

      <Step title="Open the dashboard">
        | Service | URL |
        |---------|-----|
        | Dashboard | http://localhost:8030 |
        | API docs | http://localhost:8031/docs |
      </Step>

      <Step title="Log in to collection platform accounts">
        A Chrome window opens automatically. Navigate to any platform you want to collect from (Bilibili, Twitter/X, LinkedIn, etc.) and log in normally. Your login state is saved to `~/.opencli-admin/chrome-profile/` and persists across restarts.
      </Step>
    </Steps>

  </Tab>
</Tabs>

## CLI flags (native shell)

Pass flags to `start.sh` to skip components you do not need:

```bash
./start.sh --no-chrome      # Skip Chrome — useful for RSS/API sources only
./start.sh --no-frontend    # Start backend API only, no Vite dev server
./start.sh --cdp-port 9223  # Use a custom Chrome CDP port
```

## Stopping

<Tabs>
  <Tab title="Docker">
    ```bash
    docker compose down
    ```
  </Tab>
  <Tab title="Native Shell">
    Press `Ctrl+C` in the terminal running `start.sh`. The script traps the signal and kills all child processes — Chrome, uvicorn, and Vite — automatically.
  </Tab>
</Tabs>
